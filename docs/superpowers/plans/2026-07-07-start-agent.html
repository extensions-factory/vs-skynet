Start Agent + Codex Preflight — Implementation Plan

Overview

Goal: add a Skynet: Start Agent command that launches a codex interactive session as the active session, gated by a preflight that confirms codex is installed and signed in.

Two pure, dependency-injected units. preflight.ts (checkCodex(run, codexHome), injected child-process runner) and start-agent.ts (startAgentCommand(deps), injected window/config/adapter/accessors) — mirroring the released task-handoff.ts. extension.ts supplies the real execFile runner, VSCode config, codexAdapter.runInteractive, and setActiveSession. No adapter code changes.

Expected Outcome

Working behavior

  • US-1: run Skynet: Start Agent once → a codex session launches in a terminal and becomes the active session that Send Task / Stop Agent operate on — no per-run prompts.
  • US-2: when codex is not installed / not signed in (or no folder / no home configured), the command reports a distinct actionable message and never opens a terminal.

Artifacts

  • src/adapters/codex/preflight.ts — checkCodex detection unit.
  • src/commands/start-agent.ts — startAgentCommand(deps) handler.
  • src/extension.ts — registers skynet.startAgent, wires real deps (extension.test.ts updated).
  • package.json — command and skynet.codex.* settings (activation auto-generated from the command).

How to see it working

Extension Development Host, codex installed + signed in, skynet.codex.home set to an isolated home, a folder open → Skynet: Start Agent opens a Skynet · codex … terminal that reaches readiness, then Skynet: Send Task drives a turn. Point skynet.codex.home at an empty dir → Start Agent shows "codex is not signed in. Run: codex login" and no terminal opens.

Global Constraints

  • TypeScript only; Biome only; kebab-case filenames; camelCase functions, PascalCase types.
  • Unit tests: vitest, colocated *.test.ts, headless — no real codex, no real terminal, no real child_process spawn (inject the runner).
  • pty/terminal only for orchestration; no provider SDK/API dep. The preflight is the only new child_process use, confined to install/auth diagnostics.
  • Codex home resolved as skynet.codex.home → CODEX_HOME env → error.
  • Preflight parses codex login status stdout (Not logged in), not exit code (exit 0 both ways).
  • Sandbox default workspace-write; model omitted → codex default.
