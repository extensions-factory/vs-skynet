Output — what this delivers

After all tasks land: codexAdapter.runInteractive(opts) drives a real codex TUI in a VSCode terminal through a file mailbox — multi-turn steering with pause/resume in one live session.

  • src/adapters/ — shared types.ts (worker events/usage/result; AgentAdapter added in Task 9) + pure classifyError.
  • src/adapters/interactive/ — CLI-agnostic core: mailbox, doorbell, crash-watch, session-harvester, interactive-session state machine, config.ts (central prompt strings + timing defaults), vscode terminal transport.
  • src/adapters/codex/ — codex profile (launch argv, rollout parser) + codexAdapter.

One turn: runInteractive writes protocol.md + turn-0 inbox, launches codex → turn-0 readiness handshake (re-ping once) → send() writes inbox/turn-N.md, rings doorbell, polls outbox/turn-N.json → TurnResult → sessionId/usage harvested from rollout JSONL → dispose() clears the mailbox + terminal.

CODEX_HOME — mandatory (multi-account isolation). codex reads its account/auth from CODEX_HOME. Skynet runs many agents on distinct accounts, so codexAdapter.runInteractive requires CODEX_HOME (or opts.configDir) and throws without it. Flows to the launch env (configEnv) and rollout harvest (sessionDir). This machine: CODEX_HOME=/Users/binn/.agents/codex-plus.

Central config: all prompt strings + timing defaults live in src/adapters/interactive/config.ts. Run: pnpm test (e2e skipped) · real e2e CODEX_HOME=… CODEX_INTERACTIVE_E2E=1 pnpm run test:integration.

Global constraints

  • pty/terminal only — no codex exec --json, no provider SDK. Single-turn session = fast path.
  • Launch argv (codex-cli 0.142.4): codex -C <cwd> [-m <model>] -s <sandbox> -a never -c disable_paste_burst=true -c 'tui.keymap.composer.submit="tab"' -c 'tui.keymap.composer.queue="ctrl-q"'; submit "\t".
  • Timeouts: turnTimeoutMs 300000; readyTimeoutMs 30000 (turn-0 only, re-pings once). Real turns (N ≥ 1) never re-pinged.
  • Protocol via .skynet/<workerId>/protocol.md (gitignored, disposable). Project AGENTS.md never read or written.
  • Crash detection ps -Ao pid,ppid,comm — macOS/Linux only. Mailbox reader is a ~500ms poll loop.
  • Unit (vitest): colocated src/adapters/**/<m>.test.ts, describe/test/expect. Integration (mocha): src/test/**, suite/test + node:assert, real vscode.
  • Relative imports omit .js. Real-CLI e2e gated behind CODEX_INTERACTIVE_E2E.
