Goal & approach

Goal: add two VSCode commands — send a follow-up task to the running agent, and stop it — as a thin edge layer over InteractiveSession.send() / dispose().

Seam, not UI. Two pure handlers in task-handoff.ts take an injected getSession accessor + a vscode.window-compatible HandoffWindow — no vscode import, fully unit-testable (mirrors F1.2 notify-awaiting-input.ts). The composition root holds the single session, wires vscode.window, registers both commands. No new core class.

Expected outcome

Working behavior

  • US-1: Skynet: Send Task → type a prompt → delegated via session.send(); outcome shown as a notification. Busy / no-agent refused with a message, no crash.
  • US-2: Skynet: Stop Agent → session.dispose() → status stopped; no-agent / already-stopped show a message.

Artifacts

  • src/adapters/interactive/task-handoff.ts — sendTaskCommand(), stopAgentCommand(), HandoffWindow, status guards.
  • src/adapters/interactive/task-handoff.test.ts — unit tests for both handlers.
  • src/extension.ts — holds the active session (activeSession + setActiveSession for F1.1) and registers both commands.
  • package.json — palette entries; src/test-utils/vscode-mock.ts — adds showInputBox/showErrorMessage.

How to see it working

pnpm run test:unit passes, including task-handoff.test.ts and the updated extension.test.ts. In the Extension Development Host both commands report "No running agent." until F1.1 calls setActiveSession().

Global constraints

  • TypeScript only; tabs for indentation.
  • Handler module MUST NOT import vscode — inject via HandoffWindow; only extension.ts imports vscode.
  • No new core class for the seam — call send()/dispose() directly (SC-4).
  • vitest, *.test.ts colocated; Biome (pnpm run lint) clean; Conventional Commits.
